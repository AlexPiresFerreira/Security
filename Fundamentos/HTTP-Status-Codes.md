
# 📊 HTTP Status Codes (Códigos de Status HTTP)

> **Tags:** #http #status-codes #web #rfc9110 #protocols

---

## 📌 Visão Geral

**HTTP Status Codes** são códigos de resposta padronizados que o servidor web retorna para indicar o **resultado de uma requisição HTTP**.

🎯 **Objetivo:** Comunicar ao cliente (navegador, aplicação) o que aconteceu com sua requisição.

**Referência oficial:** [RFC 9110](https://datatracker.ietf.org/doc/html/rfc9110#name-status-codes)

---

## 🔢 Categorias de Status Codes

### 1️⃣xx - Informativo (Informational)

**Descrição:** A solicitação foi **recebida** e o processo está **em andamento**.

| Código | Nome | Descrição |
|--------|------|-----------|
| **100** | Continue | Servidor recebeu cabeçalhos, cliente pode continuar enviando corpo |
| **101** | Switching Protocols | Servidor está mudando de protocolo conforme solicitado |
| **102** | Processing (WebDAV) | Servidor recebeu e está processando (evita timeout) |
| **103** | Early Hints | Servidor envia dicas sobre recursos que serão necessários |

**Uso:** Raramente visível para usuários finais.

---

### 2️⃣xx - Sucesso (Successful)

**Descrição:** A solicitação foi **recebida, compreendida e aceita** com sucesso.

| Código | Nome | Descrição |
|--------|------|-----------|
| **200** | OK | ✅ Requisição bem-sucedida (padrão) |
| **201** | Created | Recurso foi criado com sucesso (POST) |
| **202** | Accepted | Requisição aceita para processamento (assíncrono) |
| **204** | No Content | Sucesso, mas sem conteúdo no corpo da resposta |
| **206** | Partial Content | Apenas parte do recurso foi retornada (download resumido) |

**Exemplo:**

http HTTP/1.1 200 OK Content-Type: text/html Content-Length: 1234

---

### 3️⃣xx - Redirecionamento (Redirection)

**Descrição:** **Outras ações** precisam ser tomadas para concluir a solicitação.

| Código | Nome | Descrição |
|--------|------|-----------|
| **301** | Moved Permanently | ♻️ Recurso movido **permanentemente** para nova URL |
| **302** | Found (Moved Temporarily) | 🔄 Recurso movido **temporariamente** para nova URL |
| **303** | See Other | Redirecionar para outra URL usando GET |
| **304** | Not Modified | Recurso não foi modificado (cache válido) |
| **307** | Temporary Redirect | Redirecionamento temporário (mantém método HTTP) |
| **308** | Permanent Redirect | Redirecionamento permanente (mantém método HTTP) |

### 4️⃣xx - Erro do Cliente (Client Error)

**Descrição:** A solicitação contém **sintaxe incorreta** ou **não pode ser atendida**.

| Código | Nome | Descrição |
|--------|------|-----------|
| **400** | Bad Request | ❌ Requisição malformada (sintaxe inválida) |
| **401** | Unauthorized | 🔐 Autenticação necessária (falta credenciais) |
| **403** | Forbidden | 🚫 Acesso negado (sem permissão) |
| **404** | Not Found | 🔍 Recurso não encontrado |
| **405** | Method Not Allowed | Método HTTP não permitido (ex: POST em recurso GET-only) |
| **408** | Request Timeout | Cliente demorou muito para enviar requisição |
| **409** | Conflict | Conflito com estado atual do recurso |
| **410** | Gone | Recurso existiu mas foi **permanentemente removido** |
| **413** | Payload Too Large | Corpo da requisição muito grande |
| **414** | URI Too Long | URL muito longa |
| **415** | Unsupported Media Type | Tipo de mídia não suportado |
| **418** | I'm a teapot | 🫖 Easter egg (RFC 2324 - HTCPCP) |
| **429** | Too Many Requests | Rate limit excedido |
| **451** | Unavailable For Legal Reasons | Bloqueado por razões legais |
