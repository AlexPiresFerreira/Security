
# 🔍 Port Scanning (Varredura de Portas)

> **Tags:** #port-scanning #nmap #hping #netcat #tcpdump #reconnaissance #pentest

---

## 📌 Visão Geral

**Port Scanning** é a técnica de **identificar portas abertas** em um sistema para descobrir serviços em execução.

🎯 **Objetivo:** Mapear superfície de ataque de um alvo.

---

## 🚪 Portas TCP/UDP

### Categorias de Portas

**Existem 65.535 portas disponíveis, divididas em 3 categorias:**

#### 1. Portas Conhecidas (Well-known Ports)
**Range:** 0 - 1.023

**Reservadas para serviços amplamente utilizados:**
- **21:** FTP
- **22:** SSH
- **23:** Telnet
- **25:** SMTP
- **53:** DNS
- **80:** HTTP
- **110:** POP3
- **143:** IMAP
- **443:** HTTPS
- **445:** SMB
- **3306:** MySQL
- **3389:** RDP

---

#### 2. Portas Registradas
**Range:** 1.024 - 49.151

**Usadas por aplicativos registrados na IANA:**
- **1433:** Microsoft SQL Server
- **1521:** Oracle Database
- **3000:** Node.js (desenvolvimento)
- **5432:** PostgreSQL
- **8080:** HTTP alternativo (proxy, desenvolvimento)
- **8443:** HTTPS alternativo

---

#### 3. Portas Dinâmicas/Privadas
**Range:** 49.152 - 65.535

**Usadas temporariamente por aplicativos para comunicação.**

---

## 🚩 Flags TCP

As **flags TCP** controlam o estado da conexão:

| Flag | Nome | Descrição |
|------|------|-----------|
| **URG** | Urgent | Dados urgentes (processar antes da fila) |
| **ACK** | Acknowledgment | Confirma recebimento de pacotes |
| **PSH** | Push | Entregar dados imediatamente ao aplicativo |
| **RST** | Reset | Redefinir conexão (interromper) |
| **SYN** | Synchronize | Iniciar conexão (sincronizar sequências) |
| **FIN** | Finish | Encerrar conexão |

---

### Three-Way Handshake (Conexão TCP)

Cliente Servidor │ │ ├────────SYN──────────────>│ │ │ │<───────SYN/ACK───────────┤ │ │ ├────────ACK──────────────>│ │ │ └──────Conexão Estabelecida

#### Flags do Hping3

| Flag | Descrição |
|------|-----------|
| `-1` | Enviar pacotes ICMP (Ping) |
| `-2` | Enviar pacotes UDP |
| `-S` | Enviar pacotes TCP com flag SYN |
| `-A` | Enviar pacotes TCP com flag ACK |
| `-F` | Enviar pacotes TCP com flag FIN |
| `-X` | Enviar pacotes TCP Xmas (FIN, PSH, URG) |
| `-p <porta>` | Especificar porta de destino |
| `-s <porta>` | Especificar porta de origem |
| `-d <tamanho>` | Tamanho do payload em bytes |
| `-c <número>` | Número de pacotes a enviar |
| `-I <interface>` | Interface de rede |
| `--flood` | Inundar com pacotes SYN (SYN flood) |
| `-e` | Adicionar payload de texto |
| `--spoof <IP>` | IP spoofing (falsificar origem) |
| `--rand-source` | Origem aleatória |
