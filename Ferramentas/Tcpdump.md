
# 🔍 Tcpdump - Packet Analyzer

> **Tags:** #tcpdump #packet-capture #network #security #forensics #wireshark #sniffer

---

## 📌 Visão Geral

**Tcpdump** é uma ferramenta de linha de comando para **captura e análise de pacotes de rede** em tempo real.

🎯 **Objetivo:** Capturar, filtrar e analisar tráfego de rede para troubleshooting e análise de segurança.

---

## 📥 Instalação

bash
##### Debian/Ubuntu

sudo apt install tcpdump

##### Red Hat/CentOS

sudo yum install tcpdump

##### Verificar instalação

tcpdump --version

## 🔧 Opções Principais

| Opção | Descrição |
|-------|-----------|
| `-i <interface>` | Especificar interface de rede |
| `-c <número>` | Capturar apenas N pacotes |
| `-w <arquivo>` | Salvar captura em arquivo |
| `-r <arquivo>` | Ler captura de arquivo |
| `-n` | Não resolver nomes (DNS) |
| `-nn` | Não resolver nomes e portas |
| `-v` | Verbose (mais detalhes) |
| `-vv` | Muito verbose |
| `-vvv` | Extremamente verbose |
| `-X` | Exibir conteúdo em hexadecimal e ASCII |
| `-A` | Exibir conteúdo em ASCII |
| `-s <tamanho>` | Snaplen (tamanho do pacote capturado) |
| `-tttt` | Timestamp legível |

## 🏷️ Flags TCP no Tcpdump

**Tcpdump separa os protocolos por ":"**

**Exemplo de linha capturada:**
```

Flags [.], ack 1685449252, win 115, length 0 --> TCP

```
### Representação de Flags

| Flag | Símbolo | Descrição |
|------|---------|-----------|
| **SYN** | `[S]` | Synchronize - Iniciar conexão |
| **ACK** | `[.]` | Acknowledgment - Confirmar recebimento |
| **PSH** | `[P]` | Push - Entregar dados imediatamente |
| **RST** | `[R]` | Reset - Redefinir conexão |
| **FIN** | `[F]` | Finish - Encerrar conexão |
| **URG** | `[U]` | Urgent - Dados urgentes |
| **SYN+ACK** | `[S.]` | Resposta ao SYN |
| **PSH+ACK** | `[P.]` | Push com confirmação |
| **FIN+ACK** | `[F.]` | Encerramento com confirmação |

**Padrão de retransmissão:**
- 1ª retransmissão: **1 segundo**
- 2ª retransmissão: **2 segundos**
- 3ª retransmissão: **4 segundos**
- 4ª retransmissão: **8 segundos**

**Retransmissão de em razão de bloqueio** 
- A presença de flag SYN sem a flag ACK
- Números de sequencia identicos em todos os seguimentos
- Os mesmos sockets de origem e de destino
- Uma diferença de tempo, contada em dobro a cada retransmissão

RTT (Round-Trip Time)
RTO (Retransmission Time-Out)
MSL
